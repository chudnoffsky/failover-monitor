# Failover Monitor

Тестовое DevOps-задание: приложение мониторит несколько сервисов по URL с токеном
и при падении текущего лидера автоматически выбирает нового мастера (failover),
запоминая его между перезапусками.

Стек упакован в Docker, разворачивается через `docker compose`, снабжён CI/CD
(GitLab) и автоматической доставкой на сервер.

> Реализация ведётся по шагам — см. историю коммитов.
